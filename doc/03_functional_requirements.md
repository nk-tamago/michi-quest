# 03. 機能要件定義 (Functional Requirements)

## 1. アプリケーションの基本動作
*   **FR-1-1 タブ切り替え**: チャット、地図、設定の3画面をタブ等でシームレスに切り替えられること。
*   **FR-1-2 永続化**: 対話履歴、設定、APIキーなどの情報はブラウザの LocalStorage に保存され、リロードしても復元されること。

## 2. ナビゲータ（AI）とのチャット機能
*   **FR-2-1 冒頭の挨拶**: 新規ミッション（セッション）の開始時、ミッション要求を行う前に、システムからオペレーター（ミチ・ノマ）の性格を反映した初期挨拶メッセージが提示されること。
*   **FR-2-2 ミッション要求**: 初期の挨拶メッセージ表示後、システムが自動的に現在地（または設定）に基づき目的地や撮影ミッションの生成をAIへ要求し、AIが返答すること（ユーザーによる初回ボタン操作は不要）。
*   **FR-2-3 ハルシネーション対策**: AIが実在しない地名を生成するのを防ぐため、必要に応じてNominatim APIを利用して地理的に検証し、NGの場合は自動的に再生成を行うこと。
*   **FR-2-3 メッセージ通信**: 一般的なチャットアプリのように、ユーザーとAIが交互にメッセージを送信できること。
*   **FR-2-4 システムタグの解釈**: AIの応答テキストに含まれる `[AREA: {...}]` や `[TITLE: ...]` などの特殊メタ情報タグをフロントエンドでパースし、非表示にしつつアプリ側の状態に反映させること。

## 3. 画像報告と検証機能
*   **FR-3-1 画像アップロード**: デバイスのカメラまたはアルバムから写真を選択・送信できること。（※モバイル環境でもカメラが強制起動されないよう、OSネイティブの写真選択機能に対応すること）
*   **FR-3-2 リサイズと最適化**: Gemini APIへ送信する前に、クライアント側で画像を軽量（例: 長辺800px以下等）に自動リサイズすること。
*   **FR-3-3 Exif情報の抽出**: 写真からExif情報（GPS座標）を抽出し、解析プロンプトに文字列として付与できること。
*   **FR-3-4 画像判定**: 送信された画像をマルチモーダルAIが解析し、ミッションの達成条件を満たしているか、被写体が合致しているかを判定すること。

## 4. 地図・GPS機能
*   **FR-4-1 現在地取得**: デバイスのGeolocation APIを用いて現在地を取得し、地図（Leaflet）上に表示すること。ただし、ユーザーの自由な地図操作を妨げないよう自動追従（watchPosition）は行わず、地図上に配置された「現在地ボタン」を押下した際にのみ位置情報を再取得して現在地へ移動する手動更新仕様とする。
*   **FR-4-2 ミッション地点表示**: AIが指定した `[AREA: {"lat": ..., "lng": ..., "r": ...}]` に基づき、地図上に範囲をサークル等でオーバーレイ表示すること。

## 5. コレクション・ゲーミフィケーション
*   **FR-5-1 スコア機能**: ミッションの評価に応じてポイントを獲得し、累積スコアを管理できること。
*   **FR-5-2 動的生成・称号コレクション**: 獲得ポイントまたは画像評価に応じて、AIに独自の「称号」を生成させ、それをコレクションとして保存・閲覧できること。
*   **FR-5-3 ミッションクリアと演出**: 画像報告を通じて**60点以上**のスコアを獲得した場合に「ミッションクリア」と判定し、達成感のあるアニメーション演出（画面全体のスタンプや強調表示等）を再生すること。また、履歴画面にもクリア済みのバッジ等を表示すること。60点未満の場合は不合格としてやり直しを指示すること。

## 6. その他付加機能
*   **FR-6-1 キャラクター設定**: AIオペレータ（ナビゲータ）の性格、口調、感情（怒り・喜びなど）別のアバター画像を設定画面から自由にカスタマイズできること。また、初期状態としてあらかじめ用意されたデフォルト画像（通常、怒り、喜び、呆れ）が適用されること。
*   **FR-6-1-A 手動画像トリミング**: アバター画像として新しい画像をアップロードする際、または既に設定されている画像（デフォルト画像を含む）を調整する際、手動でズームや位置合わせ（クロップ）ができるUIを提供すること。
*   **FR-6-1-B 感情豊かなAI応答**: どの会話モード（ミッション生成、写真報告、通常チャット）であっても、AIは常に文脈に沿って感情豊かなテキストおよび感情タグを生成し、アバター画像の表情が頻繁に変化するようにプロンプトで制御すること。
*   **FR-6-2 リプレイモード**: 過去のチャットログを1メッセージずつ手動または自動で順送り再生でき、プレイ動画収録用に活用できること。その際、画面録画をよりリアルにするため以下の演出を行う。
    *   リプレイ専用のコントロールUIは隠蔽し、通常のチャット入力エリアを偽装表示する。
    *   ユーザー（プレイヤー）発言時は、テキストエリアへの文字を一括表示（タイピング演出はなし）し、わずかなディレイの後に送信されるようにする。
    *   AI発言時は、実際の回答時と同様に「入力中...」のディレイ演出を再現する。
    *   「自動再生モード」と「手動再生モード」が存在し、偽装された「送信ボタン」は、自動再生時は「再生停止/開始」のトグル機能、手動再生時は「次へ進む」手動送り機能として働くようにする。
    *   ミッションクリア時の全画面アニメーションなどの演出は、録画時に実際のプレイと同じタイミングで表示されるよう、履歴データにクリアしたタイミングを保持して正確に再現する。
*   **FR-6-3 データ管理**: 設定画面から、過去に獲得した称号リストや累積スコアを初期化（リセット）できること。
*   **FR-6-4 データのエクスポート**: 過去の調査履歴、獲得スコア、設定データなどローカルに保存されている状態をJSON形式のファイルとしてエクスポートできること。
*   **FR-6-5 データのインポート**: エクスポートしたJSONファイルを読み込み、現在のデータを上書き（インポート）できること。インポート時はデータフォーマットの妥当性を検証すること。また、既にローカルに履歴や設定データが存在する場合は、インポートにより既存データが完全に差し替えられ消失する旨を警告するダイアログを表示し、ユーザーの同意を得ること。
