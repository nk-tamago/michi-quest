# 01. MichiQuest 全体概要 (Overview)

## 1. プロジェクトの背景と目的
**MichiQuest（ミチ・クエスト）** は、ツーリングや散歩といった日常の移動に「ゲームのような目的と達成感」を与えることを目的としたAIナビゲーション・アプリケーションです。
生成AI（Google Gemini API）を活用して、現在地周辺の探索ミッションを動的に生成し、ユーザーが実世界で指定された場所で写真を撮影して報告することで、AIによる評価やユニークな「称号」を獲得できる体験を提供します。

## 2. ターゲットユーザー
*   目的もなく走り出しがちなバイク乗り（ライダー）、サイクリスト、自転車乗り
*   散歩やランニングに新しい刺激が欲しい人
*   AIキャラクター（ナビゲータ）とのパーソナライズされた対話を楽しみたい人

## 3. コアバリュー（提供価値）
*   **AIによる予測不能なミッション提案**: ユーザーの現在地に合わせた行き先や、特定のテーマ（例：「赤い橋」「古い看板」「道祖神」）を持ったミッションなど、毎回異なる目的を提供します。
*   **写真とGPS（Exif）を用いた判定システム**: 現地へ到達した証明として写真をアップロードし、そのExifデータ（GPS座標）と画像認識により、条件をクリアしているかAIが判定します。
*   **没入感を高めるキャラクター設定**: AIオペレーターの性格や口調（ツンデレ、軍人風、妹系など）を自由に設定し、没入感のある掛け合いを楽しめます。
*   **コレクション要素とリプレイ機能**: ミッション完了時にもらえるユニークな称号を集めたり、過去のツーリング履歴を当時のチャットの流れで再生（リプレイ）し、プレイ動画（YouTube等）の素材として活用できる機能を備えています。

## 4. 開発フェーズの変遷
*   **フェーズ1**: 基盤となるチャットUI、地図（Leaflet）連携、Gemini APIによるミッション生成と写真判定の基礎構築。
*   **フェーズ2**: 写真からのGPS（Exif）抽出、画像の自動リサイズ、キャラクターカスタマイズ（アバター画像、プロンプト）。
*   **フェーズ3**: ハルシネーション（実在しない場所の提示）対策、動的な称号コレクション機能、システム通知の会話化。
*   **フェーズ4**: 動画制作用の過去履歴「リプレイモード」機能の追加（チャット入力UIとコントローラーのシームレスな切り替え）。
