# 05. ゲーム体験リデザイン計画 (Game Experience Redesign)

## 1. 背景と目的

MichiQuestは当初「オペレーターからの指示でミッションをこなす」体験として設計されたが、キャラクター「ミチ・ノマ」の誕生に伴い「三遠地方の真理を追う探究者と、その助手であるプレイヤーの調査体験」に変遷した。

しかし現行設計には以下の課題が残っている：

-   **動機の不在**: ミチ・ノマがなぜ真理を追っているのか、何が足りないのかが語られない
-   **調査結果の断絶**: 各ミッションが単発で完結し、プレイヤーの調査結果が次の体験に繋がらない
-   **ゲーム性の浅さ**: 数値スコアと称号コレクションが「失われた知識の再構築」という世界観と噛み合っていない

本リデザインでは、**フィールドノート蓄積型**のゲーム体験に移行し、すべての報酬と評価を「知識の蓄積」に一本化する。

---

## 2. 世界観

> ミチ・ノマは、三遠地方に断片化して散らばる **「失われた知識体系」** を再構築しようとしている。  
> 橋の構造、風紋のパターン、地場の食文化、祭りの風習——あらゆる調査データは「フィールドノート」に記録され、断片が集まるほど、この地方に眠る真理の輪郭が見えてくる。  
> プレイヤー（助手）が持ち帰るすべての写真・報告が、この壮大な知識復元プロジェクトの一片となる。

### モンハン型の再訪モデル

モンスターハンターを参考に、**同じ場所に何度でも調査に行ける**設計とする。

| モンハンの要素               | MichiQuestでの対応                     |
| ---------------------------- | -------------------------------------- |
| 同じモンスターを何度も狩猟   | 同じ場所・テーマに何度でも調査可       |
| 素材が蓄積されて装備が作れる | 知見が蓄積されてフィールドノートが充実 |
| モンスターごとの図鑑が埋まる | 知見コレクションが増えていく           |
| クエスト報酬がランダム       | AIが毎回異なる視点・考察を提供         |

再訪の動機づけ：

-   同じ場所でも季節・時間帯・天候で異なる知見が得られる
-   知見の数自体がコレクション要素になる

## 3. 新ゲームメカニクス

本リデザインでは、現行のスコア・称号システムを廃止し、以下の4つのメカニクスに置き換える。

### 3.1 ★データ品質判定（スコアの代替）

数値スコアの**表示を廃止**し、提出データの学術的価値を★グレードで判定する。

| グレード | ラベル                  | 内部スコア | ゲーム的効果                                     |
| -------- | ----------------------- | ---------- | ------------------------------------------------ |
| ★★★      | 貴重データ (Rare)       | 80点以上   | 知見獲得 + ミッションクリア + 信頼度 **大幅UP**  |
| ★★☆      | 有効データ (Valid)      | 60〜79点   | 知見獲得 + ミッションクリア + 信頼度 **UP**      |
| ★☆☆      | 参考データ (Reference)  | 30〜59点   | 知見獲得（低品質）+ 未クリア + 信頼度 **微DOWN** |
| ☆☆☆      | 不適格データ (Rejected) | 29点以下   | 知見なし + 再調査指示 + 信頼度 **DOWN**          |

**ミッションクリア判定**: ★★☆（GRADE 2）以上でクリア。

**AIタグ形式**:
```
[GRADE: 3]
```
AIは内部的にスコアも算出するが、ユーザーには★グレードのみ提示する。

### 3.2 知見コレクション（称号の代替）

称号（プレイヤーへの二つ名）を**廃止**し、調査で得られた**「知見」そのものがコレクションアイテム**となる。

-   テーマを事前定義しない。**ミチ・ノマが毎回、調査結果に応じて知見のタイトルと内容を生成**
-   同じ場所でも再訪すれば**異なる視点の知見**が得られる

**AIタグ形式**:
```
[INSIGHT: {
  "title": "天竜川水系における重力式ダムの力学的美学",
  "description": "佐久間ダムの質量配分と水圧抵抗の設計思想に、明治期治水工学の到達点を確認",
  "location": "佐久間ダム"
}]
```

-   ★☆☆以上で `INSIGHT` タグが生成される（☆☆☆の不適格時は生成されない）
-   タイトルはミチ・ノマらしい学術的な表現

#### 知見獲得の2経路

1.  **指示調査（メインクエスト）**: ミチ・ノマからの調査依頼 → 写真報告 → グレード判定 + 知見獲得
2.  **自発写真（素材集め）**: 調査中にプレイヤーが良かれと思って送った写真に対し、ミチ・ノマが反応して知見を生成。**グレード判定なし**だが知見は蓄積される（信頼度にも影響：有益なら+、無関係なら-）

**データ構造** (LocalStorage: `michi-field-notes`、既存の `titlesCollection` を置き換え):
```javascript
[
  {
    id: "ins_1709369600000",
    title: "天竜川水系における重力式ダムの力学的美学",
    description: "佐久間ダムの質量配分と...",
    location: "佐久間ダム",
    grade: 3,                 // ★の数（自発写真の場合は null）
    source: "mission",        // "mission" | "spontaneous"
    sessionId: 1709369600000,
    date: "2026-03-02T16:40:00+09:00"
  }
]
```

### 3.3 信頼度システム（Trust）

ミチ・ノマの助手への「信頼度」が、プレイヤーの行動に応じて**リアルタイムに上下する**。上限・下限なし（理論上）。数値はそのまま表示されず、ミチ・ノマの態度として表現される。

#### 変動要因

| アクション           | 変動量  | 例                       |
| -------------------- | ------- | ------------------------ |
| ★★★ 貴重データ報告   | **+15** | 極めて精度の高い調査結果 |
| ★★☆ 有効データ報告   | **+8**  | 基準を満たす堅実な報告   |
| ★☆☆ 参考データ報告   | **-3**  | 中途半端な結果           |
| ☆☆☆ 不適格データ報告 | **-10** | 調査を舐めている         |
| 自発写真（有益）     | **+5**  | 計画外だが価値ある発見   |
| 自発写真（無関係）   | **-2**  | ノイズデータの提出       |
| 会話で有益な情報提供 | **+3**  | 現地の気づきや質問       |

#### 信頼度レベルと態度変化

信頼度の累積値に応じて、**プロンプトに注入するミチ・ノマの態度指示**が動的に変わる。

| 信頼度  | 態度 | プロンプト指示                             |
| ------- | ---- | ------------------------------------------ |
| 〜0未満 | 不信 | 非常に厳しく、嫌味混じりに対応             |
| 0〜19   | 警戒 | 塩対応で、能力を試すような態度             |
| 20〜49  | 標準 | 通常のキャラクター設定通り                 |
| 50〜79  | 信用 | 時折り素直な言葉が漏れる                   |
| 80〜99  | 信頼 | 不器用ながら明確な期待と信頼が滲む         |
| 100+    | 同志 | 対等な研究パートナー。ただしツンデレは健在 |

#### UI表示: ニコちゃんマーク

-   サイドバーの累計スコア表示枠を → **ニコちゃんマーク**に置き換え
-   レベル名は表示しない。**マークだけで信頼度を表現**
-   信頼度レンジに応じてマークが変化:
    -   〜0未満: 😡
    -   0〜19: 😒
    -   20〜49: 😐
    -   50〜79: 🙂
    -   80〜99: 😊
    -   100+: 🥰
-   **変動時アニメーション**: ニコちゃんマークが一瞬大きくなって切り替わる
    -   UP時: 現在のマークが弾むようにポップして新しいマークに遷移
    -   DOWN時: 現在のマークが震えて新しいマークに遷移

**データ構造** (LocalStorage: `michi-trust-score`、既存の `totalScore` を置き換え):
```javascript
// 初期値: 20（「標準」レベルからスタート）
20
```

#### プロンプトへの組み込み方

```javascript
const trustLevel = getTrustLevel(trustScore);
const trustPrompt = {
  distrust:  "助手の能力に大きな疑念を抱いています。非常に厳しく、嫌味混じりに対応してください。",
  caution:   "助手をまだ信頼していません。塩対応で、能力を試すような態度で接してください。",
  standard:  "（通常のキャラクター設定通りに振る舞ってください）",
  credit:    "助手の実力を少し認め始めています。普段は塩対応ですが、時折り思わず素直な言葉が漏れます。",
  trust:     "助手を不器用ながらも頼りにしています。厳しい指摘の中に、明確な期待と信頼が滲みます。",
  ally:      "助手を対等な研究パートナーとして認めています。ただしツンデレは変わりません。"
};
const systemPrompt = basePrompt + "\n\n【信頼度に基づく態度】\n" + trustPrompt[trustLevel];
```

### 3.4 ミッション変更への渋々対応

-   新規セッションでミッションが自動生成された後、プレイヤーが変更を要求した場合は**一方的に却下しない**
-   ミチ・ノマは最初渋るが、プレイヤーの要望を聞いた上で**渋々対応する**
-   例: *「……やれやれ。助手の分際で調査計画に口を出すとは。……ですが、現場の状況を把握しているのはあなたの方ですからね。仕方ありません、代替の調査対象を検討しましょう」*

通常チャットプロンプト（`prompt3`）に以下の指示を追加:
```
助手がミッションの変更を要望した場合は、最初は渋りつつも、
最終的にはプレイヤーの希望を聞き入れて新しい調査対象を提案してください。
その際は必ず [AREA] タグ付きで新しい目的地を出力すること。
```

## 4. 現行システムからの移行マッピング

| 現行の要素                             | 新システムでの対応                        | 備考                                 |
| -------------------------------------- | ----------------------------------------- | ------------------------------------ |
| `totalScore`（累計スコア）             | → `michi-trust-score`（信頼度）           | 数値の羅列→動的に上下する関係性指標  |
| `titlesCollection`（称号コレクション） | → `michi-field-notes`（知見コレクション） | ランダム称号→文脈のある知見          |
| `[SCORE: 85]` タグ                     | → `[GRADE: 3]` + `[INSIGHT: {...}]` タグ  | 数値非表示。★グレード+知見を出力     |
| `[TITLE: 方向音痴]` タグ               | → 廃止。知見タイトルに統合                | 知見の学術タイトルがコレクション対象 |
| `[ANNOUNCE: ...]` タグ                 | → 維持（知見獲得のアナウンス用に転用）    | 表示する内容が変わるだけ             |
| ミッションクリア演出                   | → 維持（★★☆以上でトリガー）               | 演出のラベル変更を検討               |
| 60点以上でクリア                       | → ★★☆以上（GRADE 2+）でクリア             | 内部ロジックは近似                   |
| サイドバーの累計スコア表示             | → ニコちゃんマーク表示                    | 😡😒😐🙂😊🥰                               |

## 5. 体験フロー全体像

```
【新規セッション】
  ミチ・ノマの挨拶（信頼度に応じたニコちゃんマーク表示）
    → 自動ミッション生成
      → プレイヤーがそのまま受ける or 変更を要求
        → 変更要求時: ミチ・ノマが渋々受け入れて別のミッション生成
      → プレイヤーが現地へ移動

【調査中】
  テキスト会話 → ミチ・ノマが応答（信頼度に応じた態度）
                  → 信頼度が微変動

  自発写真の送信 → ミチ・ノマが分析 → [INSIGHT] として蓄積
                    → 信頼度が変動（有益なら+、無関係なら-）

【写真報告（ミッション判定）】
  写真送信 → AIが品質判定
    → [GRADE: 3] ★★★ 貴重データ → 知見獲得 + クリア + 信頼度UP
    → [GRADE: 2] ★★☆ 有効データ → 知見獲得 + クリア + 信頼度UP
    → [GRADE: 1] ★☆☆ 参考データ → 知見獲得 + 未クリア + 信頼度DOWN
    → [GRADE: 0] ☆☆☆ 不適格    → 知見なし + 再調査 + 信頼度DOWN

【リザルト・フィードバック】
  ★★☆以上: クリア演出 + 知見追加通知
  ニコちゃんマーク変動アニメーション
```

## 6. 段階的実装フェーズ

### Phase 1: プロンプト改修 + データ構造（最小限で効果大）

1.  プロンプト改修
    -   ミッション生成プロンプト（`prompt1`）: 「失われた知識体系」の文脈を追加
    -   写真判定プロンプト（`prompt2`）: `[SCORE]`/`[TITLE]` → `[GRADE]`/`[INSIGHT]` に変更
    -   通常チャットプロンプト（`prompt3`）: ミッション変更への渋々対応、自発写真への `[INSIGHT]` 応答
    -   挨拶文に蓄積知見数への言及を追加
2.  データ構造の変更
    -   `titlesCollection` → `michi-field-notes`（知見コレクション）
    -   `totalScore` → `michi-trust-score`（信頼度）
    -   `[GRADE]` / `[INSIGHT]` タグのパースロジック（`ChatThread.jsx`）
3.  信頼度のプロンプト注入
    -   `getTrustLevel()` 関数の実装
    -   信頼度に応じた態度指示の動的生成

### Phase 2: フィールドノートUI + 信頼度ゲージ ✅

1.  ✅ 知見コレクション画面（モーダル）をアナログ革張り手帳風のデザインにリニューアル
    -   革調のダークブラウン(`#4E342E`)背景 + ステッチ風ボーダー
    -   中面はクラフト紙調 (`#FDF5E6`) + 罫線背景
    -   各知見カードはセリフフォント (`font-serif`)、★/☆ バッジ、場所・日付を表記
2.  ✅ サイドバーにニコちゃんマーク（`text-4xl`）信頼度ゲージを追加
3.  ✅ 変動時アニメーション実装（`pop-bounce` / `pop-shake`）
    -   UP時: バウンスアニメーション（scale拡大 → 縮小 → 元サイズ）
    -   DOWN時: シェイクアニメーション（左右回転 + グレースケールフィルタ）
4.  ✅ サイドバー全体のフォントサイズを拡大（text-xs → text-sm/base、スマホでの視認性向上）
5.  ✅ クリア済みセッションの表示を「CLEAR!」バッジから吹き出しアイコン上のチェックマーク(`CheckCircle`)に変更

### Phase 3: 品質判定UIの演出改修 ✅

1.  ✅ ★グレード表示のリザルトカード化
    -   ダークレザー調カード（`bg-gradient-to-br from-[#4E342E] to-[#3E2723]`）
    -   上部にゴールドグラデーションライン
    -   「ミチ・ノマの評価」ヘッダー + Lucide `Star` アイコン（48px）×3の点灯式
    -   知見タイトル表示（`font-serif text-xl`）
2.  ✅ 自発写真からの知見獲得時に「Spontaneous discovery!」バッジをチャット内に表示
3.  ✅ 挨拶文の動的化（蓄積知見数 `{{insightCount}}` の反映）
4.  ✅ `prefers-reduced-motion` メディアクエリ対応（Web Interface Guidelines準拠）

### Phase 4: 対話体験の向上（段階的評価とスタンプ演出） ✅

1.  ✅ 写真判定結果の「段階的」開示化（ガチャ的演出）
    -   一気に結果を表示せず、まずAIから「……っ！これは……」等の「前振り（PRELUDE）」発言のみを生成・表示
    -   ユーザーに「ミチ・ノマの話を聞く」というアクションボタンを提示
    -   ボタン押下後、評価カードUIと詳細な労いコメントを時間差（1秒ディレイ）で表示
2.  ✅ クリア演出のチャット統合（スタンプUI）
    -   旧来の全画面「MISSION CLEARED!」画面を廃止
    -   チャットの延長として、AIの労いコメントの隣にCSS+SVGベースの「APPROVED（調査完了）」かすれ赤スタンプがアニメーションで押される演出に変更
3.  ✅ リロード・リプレイ時のUI復元機能
    -   チャット履歴の裏データ（`pendingData`）に評価結果を隠し持つことで、リロードやリプレイ時にも的確に「話を聞く」ボタンを復元表示させる堅牢化

### Phase 5: 感情表現の拡充とインライン・エモーション ✅

1.  ✅ 感情パターンの追加（4種 → 8種）
    -   既存: `normal`, `angry`, `joy`, `disgust`
    -   新規: `blush`（照れ）, `sparkle`（興奮）, `stare`（ジト目）, `sad`（悲しみ）
2.  ✅ インライン・エモーションの実装
    -   テキストストリーミング中に `[Emotion: xxx]` タグが出現した瞬間に、アバターの表情画像を動的に切り替える仕組み。
    -   リプレイモード（手動/自動両方）においても、当時の表情変化のタイミングを再現する。
3.  ✅ プロンプトの感情トリガー強化
    -   より自然で多様な感情指定を生成させるため、プロンプトでの指示を強化。

## 7. 今後の体験向上アイデア（未実装・検討事項）

Phase 4の実装検討時に挙がったが、一旦見送った以下のアイデアは今後のアップデートにおいて実装を検討する。

### 提案A: クリア後のシームレスな後続ミッション提案
ミッションクリア後（リザルト確認後）に、AIから能動的に**「本日の調査は終了しますか？ それとも次の地点を検索しますか？」**と問いかける、あるいはボタン（Quick Reply）のようなUIを表示して、次のミッションへの導線を作ることで、体験が途切れずに継続できるようにする。

### 提案B: セッション開始時の「意向確認」フェーズの導入
初回時の自動ミッション要求フローを少し緩和し、初期挨拶時に**「行き先の希望はありますか？ なければ【おまかせ】と返答してください」**のようにワンクッション置き、ユーザーのテキスト送信をトリガーにしてミッション生成を行う形にする。

### 提案C: チャット上部への「ステータスバナー」追加
ユーザーが今何をすべきかを常に明確にするため、チャット画面のヘッダー下部などに「📍現在の目標：〇〇の調査」「✅ 調査完了（フリー探索中）」といった永続的なステータス表示領域を追加する。
